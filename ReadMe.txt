על מנת להפעיל את התוכנה יש להריץ את קובץ ה run.bat ולוודא כי java פונה לגרסה שהיא 64 bit

בעת עליית התוכנית קיימות מספר אפשרויות:
1. טעינת קובץ ה corpus, בחירת נתיב לשמירת קבצי הפוסטינג, ובחירת ביצוע סטמינג בעת האינדוקס
2. טעינת המילון והצגתו
3. Reset
4. הרצת שאילתא אחת שכתבנו בתוכנית
5. הרצת קובץ שאילתות שמכניסים לפי נתיב שבוחרים
6. בחירה אם לכתוב את התוצאות לבדיקת בתוכנת trec-eval בקובץ טקסט
7. חלון להצגת המסמכים שאחזרנו לכל שאילתא וכפתור להצגת 5 היישיות הדומיננטיות בכל מסמך

מקרה 1: טעינת קובץ הקורפוס לזיכרון:
	1. יש להזין נתיב לתקייה בה נמצאים כל קבצי הcorpus וקובץ הstop_words.
	2.יש להזין נתיב לתקייה בה ישמרו כל קיבצי הposting והמילונים.
	3. המשתמש יוכל לבחור האם לבצע Stemming באמצעות כפתור check box בשם Stemming (אופציונאלי)
	4. לחיצה על run תפעיל את התוכנית ותגרום לקריאת הקבצים, פירסור, וכתיבה לPosting ולמילון.
	**בעת לחיצה על run ללא הזנת הנתיבים, תוצג הודעת שגיאה וגם אם לא הוזמנו כל הנתיבים או שאינם נכונים תוצג הודעת שגיאה.
	
	לאחר סיום אינדוקס של כל הקוד תוצג למשתמש הודעה המכילה את גודל המאגר, מספר המילים הייחודיות בו, ואת הזמן שנדרש לביצוע פעולה זו (בשניות).

מקרה 2: טעינת המילון והצגתו:
	אופציה 1: לאחר שהמשתמש טען את הcorpus לחיצה על loadDictionaty טוענת את המילון לזיכרון.
		לחציה על כפתור showDicrionary תציג למשתמש את המילון.
	אופציה 2: המשתמש מפעיל את התוכנית ומיד לאחר מכן רוצה להציג את המילון.
		על המשתמש להזין נתיב בPosting שממנו הוא רוצה לטעון את המילון
		המשתמש יוכל לבחור האם לטעון מילון שעבר Stemming באמצעות כפתור check box בשם Stemming (אופציונאלי).
		המשתמש ילחץ על כפתור loadDictionaty או showDicrionary .
		**אם הנתיב לא חוקי או שלא קיים מילון בתקייה המערכת תציג הודעת שגיאה.

מקרה 3: Reset
	1. לחיצה על כפתור הreset תגרום למחיקת הקבצים בנתיב הposting ולאתחול הזיכרון של התוכנית.
	ניתן לבצע פעולה זו לאחר טעינת הcorpus או לאחר עליית המערכת בתנאי שמוזן נתיב בPosting.

מקרה 4:	הרצת שאילתא אחת שכתבנו בתוכנית:
המתשמש כותב את השאילתא שהוא רוצה לחפש במנוע במקום המיועד ואז יש ללחוץ על כפתור Run Query
המשתמש יכול לבחור האם לבצע semantic על השאילתא וגם אם לבצע סמנטיקה offline ללא חיבור לאינטרנט או online עם חיבור לAPI.

מקרה 5: הרצת קובץ שאילתות שמכניסים לפי נתיב שבוחרים:
המתשמש יכול להזין קובץ שאילתות שהוא רוצה לחפש במנוע בעזרת לחיצה על הכפתור Browse Query File לבחור את הנתיב לקובץ שרוצים להפעיל ואז יש ללחוץ על כפתור Run Query File
המשתמש יכול לבחור האם לבצע semantic על הקובץ שאילתות וגם אם לבצע סמנטיקה offline ללא חיבור לאינטרנט או online עם חיבור לAPI.

מקרה 6: בחירה אם לכתוב את התוצאות לבדיקת בתוכנת trec-eval בקובץ טקסט:
המשתמש יכול לבחור אם הוא רוצה לכתוב את התוצאות שיוצאות לקובץ טקסט שאותו יוכל להריץ בבדיקת התוכנה trec_eval. עליו ללחוץ על כפתור הTrec-Eval ולבחור את הנתיב שבו הוא רוצה לשמור את הקובץ ואת שם הקובץ.

מקרה 7: חלון להצגת המסמכים שאחזרנו לכל שאילתא וכפתור להצגת 5 היישיות הדומיננטיות בכל מסמך:
לאחר הרצת מקרה 4 או 5 יוצגו תוצאות האיחזור בחלונית ששם ניתן לראות את מזהה המסמך ולכל מזהה כזה יש כפתור שניתן ללחוץ עליו כדי לראות את חמשת היישיות הדומיננטיות למסמך.
בנוסף בצד שמאל באמצע ליד תגית הQuery# תוצג מספר השאילתא שמסמכים המוצגים בחלונית רלוונטים לגביה. כדי לדפדף בין מספרי השאילתות יש ללחוץ על כפתורי הNext והPrev
